About UserManager.Web

Before running this application in your local development environment, please check you have the required dependencies installed.

Dependency Information

This application was written using Visual Studio 2019 Community Edition version 16.3.9

The web client is built using Angular 8 (build 8.3.19) which is included with Visual Studio but requires Node.js version 12.13 to be installed.

If you have the above version of Visual Studio and Node.js installed, all you need to do is load, build and run the application.

The first time you build the application, Nuget will automatically install the required packages and this can take a few minutes to complete.

Project Architecture

The project uses the ASP.Net Core 3.0 Web Application template for Angular 8. This allows the Angular client and the server Web API to reside in the same project which is ideal for prototype applications like this.

The server is a standard REST service using ASP.Net Web API and Entity Framework Core for data persistence. Communication between the client and server is performed via HTTPClient using JSON as the payload.

SQLite has been used as the database provider, this is a local file database so it isn't necessary to have a database server or to run any SQL scripts. Code migrations can be run using Update-Database in the Package Manager Console if you need to recreate and reseed the database.

The client uses a service layer to handle all HTTPClient calls. There is an option to "Show Service Messages" in the UI so you can view the calls to the server and any exceptions returned.

The web pages and components use Bootstrap 4.3 for all styling. This comes built-in with the ASP.Net Core 3.0 project and allows a high degree of browser and mobile device support.

The Angular components are built using standard HTML5 controls and do not require any third party CSS styles or JavaScript libraries.

Design Decisions

The app has been configured to run over HTTPS. Even though this app isn't handling any 'real' user data most browsers will assume it is and display warnings otherwise.

There are specific appsetting."environment".json files included for each environment which should be used for any connection strings or config info. The webserver will only use the appropriate .json file based on it's ASPNETCORE_ENVIRONMENT setting. This way you can deploy the same solution to any environment without having to change it and only the correct config will be used.
